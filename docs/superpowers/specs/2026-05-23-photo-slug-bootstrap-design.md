# Photo-Post Slug Bootstrap Design

## Context

Every photo post currently has a slug of the shape `YYYY-MM-DD-<city>` (e.g. `2018-11-24-san-francisco`), with `-2`/`-3` suffixes when more than one photo was taken in the same city on the same day. Combined with the permalink config `posts = "/:year/:month/:slug/"`, the URL repeats the date and barely identifies the photo:

```
/2018/11/2018-11-24-san-francisco/
/2018/11/2018-11-24-san-francisco-2/
/2018/11/2018-11-24-san-francisco-3/
/2018/11/2018-11-24-san-francisco-4/
```

The archive now has much richer per-post metadata (title, alt text, tags, artists, descriptions) than when the slugs were generated by the ingest script. That metadata makes editorially-distinctive slugs feasible without any automation. This is a one-off bootstrap pass to set good slugs across all 109 existing posts before they accumulate inbound links.

The site is soft-launched (deployed to `smallobservations.net` but barely indexed and not yet shared), so the cost of breaking the current URLs is small in this single moment and large at every later moment. Bootstrap now, leave automation for later.

## Goal

Set every photo post's `slug:` field to a human-memorable string of the shape `<city>-<editorial-hook>` that:

1. Identifies the photo from the URL alone (no need to click).
2. Anchors location (city), since the URL otherwise contains only the date.
3. Reads well in browser tabs, social cards, RSS items, and search results.
4. Stays unique within its year+month bucket without resorting to `-2` suffixes wherever possible.
5. Is derivable from existing front-matter (`title`, `alt`, `tags`, `artists`, `cities`) ŌĆö no external knowledge.

## Non-goals

- Automating slug generation for future ingest. The ingest script keeps producing the current `YYYY-MM-DD-<city>` shape; this spec rewrites only the 109 existing files.
- Changing the permalink config (`/:year/:month/:slug/` stays).
- Changing the `cities[]` taxonomy values. Slug normalization is for slug use only; taxonomy URLs continue to be `/cities/greater-london/`.
- Adding aliases for old URLs. The soft-launch state makes that unnecessary for this pass. (See [Future Changes](#future-changes) below ŌĆö subsequent slug changes will not get the same dispensation.)
- Generating `description:`, `alt:`, or any other field. Those are already authoritative and supply the source material for the slug.

## Slug shape

```
<city>-<editorial-hook>
```

- `<city>` is mandatory and always comes first.
- `<editorial-hook>` is 2ŌĆō5 lowercase words separated by hyphens, drawn editorially from the post's own front matter.
- The two halves are joined with a single hyphen.
- All lowercase; only `[a-z0-9-]` characters allowed.

Target length 25ŌĆō50 characters; hard cap 70.

## City normalization

The slug prefix uses the recognisable short city name, not the administrative `cities[]` value. Front matter and taxonomy URLs are unchanged.

| `cities[]` | Slug prefix |
|---|---|
| Greater London | `london` |
| Basingstoke and Deane | `basingstoke` |
| Reykjavikurborg | `reykjavik` |
| Derbyshire Dales | `derbyshire-dales` |
| Mountain View | `mountain-view` |
| Penrhyndeudraeth | `penrhyndeudraeth` |
| Bucharest | `bucharest` |
| New York | `new-york` |
| San Francisco | `san-francisco` |
| (any other) | already-recognisable, kebab-case the existing value |

When a future post adds a new city not in this table, add a row at the time the post is ingested.

## Editorial hook rules

The hook is the part of the slug that names the photo. It is chosen per post, but the choice is constrained by the following rules so the archive stays coherent.

### Selection priority

For each post, pick *one* of these as the hook's anchor (not stacked):

1. **Named subject** ŌĆö the most famous identifiable thing in the photo. Person, animal, band, named landmark. Examples: `kurt-cobain`, `picasso`, `ringo-starr`, `beatles`.
2. **Quote-fragment** ŌĆö the most distinctive piece of text on the artwork, when the text is the post. Examples: `hella-resist`, `tangerines-are-better`, `war-is-over`.
3. **Named artist + brief subject** ŌĆö when the maker is the strongest hook. Examples: `niall-olochlainn-yes`, `banksy-child-with-mallet`, `nathan-bowen-stay-positive`.
4. **Named theme** ŌĆö a recognised movement, event, or campaign already in `tags`. Examples: `marriage-equality`, `extinction-rebellion`, `palestine-watermelon`.
5. **Visual hook** ŌĆö a short visual descriptor from `alt`, used when nothing more specific exists. Examples: `pink-flamingo`, `vampire`, `brown-bird`.

Choose the highest-priority anchor that the post genuinely has. A post with both a famous subject and an artist signature usually leads with the subject; the artist may still appear inside the hook if it sharpens the slug.

### Grounding

Every hook word must derive from one of the post's own fields: `title`, `alt`, `tags`, `artists`, `cities`, or body text. No external knowledge. This mirrors the rule already in `2026-05-17-titles-descriptions-design.md` and the descriptions amendment.

### Drop rules

These are removed from the hook even when present in the source field:

- **Articles and prepositions:** `a`, `an`, `the`, `of`, `on`, `in`, `at`, `with`, `by`, `from`, `to`, `and` (unless joining is genuinely meaningful, e.g. `marriage-equality` keeps no `and`; `house-keys-not-handcuffs` keeps `not` because it carries the slogan).
- **Generic form nouns:** `mural`, `stencil`, `figure`, `painting`, `installation`, `sculpture`, `portrait`, `sticker`, `sign`, `scene`. Drop unless removing them makes the slug ambiguous (rare).
- **Generic adjectives:** colour words, size words, shape words ŌĆö only when they don't carry identifying weight. A `pink-flamingo` keeps `pink`; a `bright-orange-monster-face` becomes just `orange-monster`.
- **Punctuation:** quotes, apostrophes, em-dashes, exclamation marks, commas, full stops.

### Transliteration

Diacritics, non-Latin characters, and special letters are transliterated to ASCII:

- `J├╝rgen` ŌåÆ `jurgen`
- `├×eim` ŌåÆ `theim`
- `├×` ŌåÆ `th`, `├░` ŌåÆ `d`, `├Ė` ŌåÆ `o`, `├¦` ŌåÆ `c`, `┼×`/`┼¤` ŌåÆ `s`, `├ü`/`├Ī` ŌåÆ `a`, `├ē`/`├®` ŌåÆ `e`
- `Bucure┼¤ti` ŌåÆ `bucuresti`
- `O'Lochlainn` ŌåÆ `olochlainn`

Numbers stay as digits: `Zap Comix No. 7` ŌåÆ `zap-7`.

### Length

- Target 25ŌĆō50 characters total slug.
- Hard cap 70 characters total.
- If a slug exceeds the cap, drop the least essential hook word.

## Disambiguation

If two posts in the same `<year>/<month>` resolve to the same slug:

1. **Preferred:** vary the editorial hook on the later post. Pick a different anchor or add one more distinguishing word. The whole point of editorial slugs is that two photos of different things should not need numeric suffixes.
2. **Last resort:** append `-2`, `-3` to the later post's slug. Document this in a comment if the disambiguation is non-obvious.

There is no requirement that slugs be globally unique across the whole archive ŌĆö Hugo only requires uniqueness within a permalink scope, which is year+month here.

## Future changes

Once the bootstrap pass lands, the URLs are *canonical*. Any subsequent slug change to a post that is already published must add an `aliases:` front-matter entry to keep the old URL working:

```yaml
aliases: ["/2018/11/san-francisco-hella-resist/"]
```

Hugo emits a small HTML redirect for each alias entry. This rule applies to renames, typo fixes, and any future editorial reconsideration. The only exception is this initial bootstrap, justified by the site's soft-launched, barely-indexed state.

This rule should also be added to `CLAUDE.md` as part of the implementation so future agents (and future-me) don't redo the breakage.

## Source discipline (carried from prior specs)

Every hook word must be defensible against the post's own data. The "no external knowledge" rule from `2026-05-17-titles-descriptions-design.md` applies in full. Saying `dublin-pride` for the Marriage Equality post would be wrong (`pride` isn't in any field); `dublin-marriage-equality` is fine (it's the title and a tag).

## Verification

After the bootstrap is applied:

1. Every photo post's `slug:` is set and matches the `<city>-<hook>` shape.
2. Every slug is lowercase `[a-z0-9-]+`, no double hyphens, no leading or trailing hyphen.
3. No slug exceeds 70 characters total.
4. No two posts in the same `<year>/<month>` share a slug.
5. `make build` passes.
6. `make check` passes (htmltest in particular catches accidentally broken internal links).
7. Spot-check 10 random new URLs in a browser to confirm they render.

## Out of scope

- Designing an ingest-time slug generator. The ingest pipeline keeps producing the date-city shape, and new posts get their editorial slug as a hand edit before publishing. A future refactor of the ingest script could automate the editorial hook from alt + tags, but that's a separate design.
- Migrating taxonomy URLs (`/cities/greater-london/` etc.) to use short names. Out of scope; taxonomy slugs are tied to the `cities[]` field.
- Aliases for the old slugs. (See above.)
