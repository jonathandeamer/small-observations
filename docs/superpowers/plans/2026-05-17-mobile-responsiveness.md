# Mobile Responsiveness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve mobile reading and browsing comfort across 320px phones, modern phones, the existing 640px gallery breakpoint, and small tablets without changing the site's small-web character.

**Architecture:** Keep the work in the existing single stylesheet, `themes/notebook/assets/css/site.css`. Add a small number of targeted media-query rules for dense components rather than rewriting templates or creating a separate mobile layout.

**Tech Stack:** Hugo templates, hand-written CSS, responsive images already generated by Hugo's image pipeline, `make build` for verification.

---

## File Structure

- Modify: `themes/notebook/assets/css/site.css`
  - Add tablet and phone refinements for gallery density, post metadata, browse rows, terms lists, pagination, and the narrowest page shell spacing.
  - Keep existing desktop rules intact.
  - Keep all responsive behaviour in the same stylesheet.

No templates should change. If a template issue is discovered during viewport testing, stop and document the finding before editing templates.

## Task 1: Add Targeted Responsive CSS

**Files:**
- Modify: `themes/notebook/assets/css/site.css`

- [ ] **Step 1: Inspect the current responsive section**

Run:

```bash
sed -n '190,245p' themes/notebook/assets/css/site.css
```

Expected: output includes `.gallery-grid`, the existing `@media (max-width: 640px)` rule, and `.browse`.

- [ ] **Step 2: Replace the existing gallery-only mobile media query with the full responsive block**

In `themes/notebook/assets/css/site.css`, replace this block:

```css
@media (max-width: 640px) {
  .gallery-grid { grid-template-columns: repeat(2, 1fr); }
}
```

with this block:

```css
@media (max-width: 760px) {
  .gallery-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 0.85rem;
  }

  .browse a {
    grid-template-columns: 2rem 1fr;
    column-gap: 0.75rem;
    row-gap: 0.1rem;
  }

  .browse .count {
    grid-column: 2;
    font-size: 0.85rem;
  }
}

@media (max-width: 640px) {
  .gallery-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 0.75rem;
  }

  .post-meta {
    grid-template-columns: 1fr;
    row-gap: 0.12rem;
    margin-bottom: 1.25rem;
  }

  .post-meta .row {
    display: grid;
    row-gap: 0.05rem;
  }

  .post-meta .row + .row {
    margin-top: 0.45rem;
  }

  .post-meta dt {
    font-size: 0.78rem;
  }

  .gallery-meta {
    gap: 0;
  }

  .gallery-place {
    font-size: 0.84rem;
    line-height: 1.25;
  }
}

@media (max-width: 380px) {
  .page {
    padding-inline: 1rem;
  }

  .masthead {
    margin-bottom: 1.1rem;
  }

  .browse a {
    grid-template-columns: 1.6rem 1fr;
    gap: 0.1rem 0.6rem;
    padding-block: 0.75rem;
  }

  .browse .label {
    font-size: 1.2rem;
  }

  .terms a {
    align-items: flex-start;
    flex-direction: column;
    gap: 0.05rem;
  }

  .pagination {
    flex-wrap: wrap;
    gap: 0.45rem 0.8rem;
  }
}
```

- [ ] **Step 3: Run a targeted CSS search**

Run:

```bash
rg -n "@media|max-width: 760px|max-width: 640px|max-width: 380px|post-meta|browse|terms|pagination|gallery-grid" themes/notebook/assets/css/site.css
```

Expected: output shows the three responsive bands and the affected components. There should be only one `@media (max-width: 640px)` block.

- [ ] **Step 4: Commit the CSS change**

Run:

```bash
git add themes/notebook/assets/css/site.css
git commit -m "style(css): improve mobile responsiveness"
```

Expected: commit succeeds. If sandboxing prevents `.git` writes, rerun the same commit command with escalation.

## Task 3: Concrete Follow-Up Adjustments For Viewport Failures

**Files:**
- Modify: `themes/notebook/assets/css/site.css`

Run this task when Task 2 records a concrete viewport problem. Skip this task when Task 2 records no viewport problems.

- [ ] **Step 1: Identify the failing viewport and component**

Record the issue in this format:

```text
Viewport: 320px
Page: /tags/
Component: terms list
Problem: long term names and counts still crowd each other.
```

Expected: one specific problem, page, viewport, and component per adjustment.

- [ ] **Step 2: Make the smallest CSS adjustment**

Use only the affected media query and component. Examples:

```css
@media (max-width: 380px) {
  .terms .term-count {
    font-size: 0.8rem;
  }
}
```

or:

```css
@media (max-width: 640px) {
  .post-meta dd {
    line-height: 1.35;
  }
}
```

Expected: no new selectors outside the existing component set unless the observed issue requires one.

- [ ] **Step 3: Re-run build**

Run:

```bash
make build
```

Expected: build exits 0.

- [ ] **Step 4: Re-check only the affected viewport and page**

Use the same browser viewport that exposed the issue.

Expected: the specific issue is resolved without creating horizontal scrolling or text clipping elsewhere on the same page.

- [ ] **Step 5: Commit the adjustment**

Run:

```bash
git add themes/notebook/assets/css/site.css
git commit -m "style(css): tune mobile layout"
```

Expected: commit succeeds. If sandboxing prevents `.git` writes, rerun the same commit command with escalation.
