# Small Observations — Hugo Site Implementation Plan (Plan A)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Hugo site — templates, partials, CSS, self-hosted typography, and responsive image pipeline — and prove it works end-to-end against 3 hand-crafted sample posts. The Python ingest script is a separate plan (Plan B) and is *not* in scope here.

**Architecture:** A theme-less Hugo site. All layouts live directly under `layouts/`. A single hand-written stylesheet under `assets/css/` is inlined into `<head>` by Hugo's resource pipeline. One self-hosted variable font (Fraunces) is preloaded. The `partials/photo.html` partial drives Hugo's `image.Process` to emit responsive `<picture>` elements (AVIF + WebP + JPEG, three widths). Taxonomies: `tags`, `countries`, `cities`, `artists`, `favourites`, `years`. The repo already has a Hugo scaffold, `.gitignore`, and a strict conventional-commit hook in `.githooks/commit-msg`.

**Tech Stack:** Hugo (extended, ≥0.128 for `image.Process`), HTML, CSS, no JavaScript. One variable woff2 font file.

**Verification philosophy:** Hugo templates don't fit unit-test TDD cleanly. Each task is verified behaviorally: start `hugo server`, fetch the rendered page with `curl`, grep for expected content, confirm no Hugo errors in the server log. This gives a real pass/fail signal per task without inventing a testing framework that isn't worth the weight on a site this small.

**Commit convention:** Every commit must pass `.githooks/commit-msg`. Format `type(scope): subject`. Allowed types: `content | feat | fix | style | refactor | docs | chore | build`. Allowed scopes: `home | post | list | partial | css | font | taxonomy | ingest | photo | note | config | spec | mockup | deploy`. Subject lowercase, no trailing period, ≤72 chars.

## Task 8: Build the responsive photo partial

This partial is the workhorse of the site. Given a post page, it locates the photo in `assets/img/`, generates AVIF + WebP + JPEG at three widths via Hugo's image pipeline, and emits a `<picture>` element with explicit `width`/`height` to prevent CLS.

**Files:**
- Create: `layouts/partials/photo.html`
- Modify: `assets/css/site.css` — append photo rules

- [ ] **Step 1: Create the partial**

Create `layouts/partials/photo.html`:

```html
{{/*
  Renders a responsive <picture> for a post's photo.
  Reads `.Params.photo` (path relative to assets/img/) and `.Params.alt`.
  Above-the-fold callers pass `eager` in the params dict to skip lazy-loading.

  Usage:  {{ partial "photo.html" (dict "page" . "eager" false) }}
*/}}

{{ $page := .page }}
{{ $eager := .eager | default false }}
{{ $rel := printf "img/%s" $page.Params.photo }}
{{ $orig := resources.Get $rel }}

{{ if not $orig }}
  {{ errorf "photo not found in assets/: %s (post: %s)" $rel $page.RelPermalink }}
{{ end }}

{{ $widths := slice 600 1000 1600 }}

{{ $jpeg600  := $orig.Resize "600x q82" }}
{{ $jpeg1000 := $orig.Resize "1000x q82" }}
{{ $jpeg1600 := $orig.Resize "1600x q82" }}
{{ $webp600  := $orig.Process "resize 600x webp q80" }}
{{ $webp1000 := $orig.Process "resize 1000x webp q80" }}
{{ $webp1600 := $orig.Process "resize 1600x webp q80" }}
{{ $avif600  := $orig.Process "resize 600x avif q60" }}
{{ $avif1000 := $orig.Process "resize 1000x avif q60" }}
{{ $avif1600 := $orig.Process "resize 1600x avif q60" }}

<picture class="photo">
  <source type="image/avif"
          srcset="{{ $avif600.RelPermalink }} 600w, {{ $avif1000.RelPermalink }} 1000w, {{ $avif1600.RelPermalink }} 1600w"
          sizes="(max-width: 700px) 100vw, 700px">
  <source type="image/webp"
          srcset="{{ $webp600.RelPermalink }} 600w, {{ $webp1000.RelPermalink }} 1000w, {{ $webp1600.RelPermalink }} 1600w"
          sizes="(max-width: 700px) 100vw, 700px">
  <img src="{{ $jpeg1000.RelPermalink }}"
       srcset="{{ $jpeg600.RelPermalink }} 600w, {{ $jpeg1000.RelPermalink }} 1000w, {{ $jpeg1600.RelPermalink }} 1600w"
       sizes="(max-width: 700px) 100vw, 700px"
       width="{{ $jpeg1600.Width }}" height="{{ $jpeg1600.Height }}"
       alt="{{ $page.Params.alt | default "" }}"
       loading="{{ if $eager }}eager{{ else }}lazy{{ end }}"
       decoding="async">
</picture>
```

## Task 17: Self-host Fraunces and preload it

Replace the implicit serif-stack fallback by self-hosting Fraunces as a variable woff2 file. Add a `@font-face` declaration and a `<link rel="preload">` so the font starts loading before CSS is parsed.

**Files:**
- Create: `assets/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2`
- Modify: `assets/css/site.css` — prepend `@font-face`
- Modify: `layouts/_default/baseof.html` — add `<link rel="preload">`

- [ ] **Step 1: Download Fraunces variable font**

Fraunces is hosted on GitHub. Download the variable woff2 file:

```bash
mkdir -p assets/fonts
curl -L -o 'assets/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2' \
  https://github.com/undercase/fraunces/raw/main/fonts/variable/Fraunces%5Bopsz%2Cwght%2CSOFT%2CWONK%5D.woff2
```

Verify with: `ls -la 'assets/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2'`. File should be ~150–250kb.

If the URL above 404s, fall back to: download Fraunces from https://fonts.google.com/specimen/Fraunces (use a tool like google-webfonts-helper to get the woff2), and place it at the same path. Italic axis is separate (`Fraunces-Italic[opsz,wght,SOFT,WONK].woff2`) — skip italic for now; the body italic will fall back to a synthesized oblique, which is acceptable for the initial build.

- [ ] **Step 2: Prepend `@font-face` to `site.css`**

Insert at the very top of `assets/css/site.css` (above `:root`):

```css
@font-face {
  font-family: "Fraunces";
  font-weight: 100 900;
  font-style: normal;
  font-display: swap;
  src: url("/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2") format("woff2-variations"),
       url("/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2") format("woff2");
  /* note: the woff2-variations format is the modern declaration;
     the second src line is a fallback for older browsers. */
}
```

- [ ] **Step 3: Publish the font to `/fonts/` and preload it**

Hugo serves files in `assets/` only when they're processed; raw files belong in `static/` to be served as-is. Move the font to where Hugo will serve it:

```bash
mkdir -p static/fonts
mv 'assets/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2' 'static/fonts/Fraunces[opsz,wght,SOFT,WONK].woff2'
rmdir assets/fonts
```

Now modify `layouts/_default/baseof.html` to add the preload tag. Find the block:

```html
  {{ block "head" . }}{{ end }}
</head>
```

And replace with:

```html
  <link rel="preload"
        href="/fonts/Fraunces%5Bopsz,wght,SOFT,WONK%5D.woff2"
        as="font"
        type="font/woff2"
        crossorigin>

  {{ block "head" . }}{{ end }}
</head>
```

(The square brackets in the filename must be URL-encoded as `%5B` and `%5D` in the `href`.)

```bash
curl -s -o /dev/null -w "%{http_code}\n" "http://127.0.0.1:1313/fonts/Fraunces%5Bopsz,wght,SOFT,WONK%5D.woff2"
```

Expected: `200`.
